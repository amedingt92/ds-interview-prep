# Capstone — End-to-End Data Science Assessment Simulation

> **Estimated time:** 60–90 minutes  
> **Run:** `python capstone/capstone.py`  
> **This is your final exam before the real one.**

---

## What This Is

The capstone simulates the style and difficulty of the actual assessment. It covers all four domains — SQL, Python, Pandas, and Snowflake concepts — in one integrated session.

Unlike the individual lessons:
- No hints are provided for most questions
- Problems are phrased the way a real assessment would phrase them (brief, outcome-focused)
- The checker will tell you pass/fail but not *why* — you need to debug your own answers
- There is a time target: aim to complete everything in under 60 minutes

---

## Structure

| Section | Questions | Domain |
|---------|-----------|--------|
| A | 5 | SQL (via DuckDB) |
| B | 5 | Python |
| C | 5 | Pandas |
| D | 5 | Snowflake concepts (written answers — self-grade) |

---

## Tips Before You Start

- Read each problem statement fully before writing code
- The checker is strict — column names and data types matter
- If you're stuck for more than 5 minutes on one question, move on and come back
- Snowflake Section D is self-graded — be honest with yourself
- After finishing, check your answers against `capstone/answers.py`

---

## Scoring Guide

| Score | Readiness |
|-------|-----------|
| 18–20 | Excellent — ready for the assessment |
| 15–17 | Good — review any sections you missed |
| 12–14 | Fair — revisit the lessons for your weak areas |
| < 12  | Spend more time on lessons before the test |

The real test requires 70% (42/60). This capstone uses 20 questions to give you a comparable signal.
